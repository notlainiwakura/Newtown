## 5. Structural Testing Gaps

### A. No integration tests with real HTTP server
- All web API tests use mocked request/response objects or source-level analysis
- No test starts a real Express server and makes HTTP requests to it
- The `startDoctorServer()` function is completely untested

### B. No end-to-end loop execution tests
- Background loop tests verify start/stop/config but never run a full cycle with a real (or mock) LLM
- No test verifies that diary loop actually calls provider.complete() and stores the result
- No test verifies that commune loop initiates a real multi-turn conversation

### C. No cross-character interaction tests
- Letter delivery between characters is never tested end-to-end
- Dream seed exchange between characters is never tested
- No test verifies that two character servers can communicate via their APIs

### D. No database migration end-to-end tests
- Palace migration and KG migration scripts are completely untested
- No test verifies that migration from an old schema version works correctly

### E. No long-running stability tests
- No test runs a loop for multiple cycles to check for timer/memory accumulation
- No test verifies that stopped loops don't leak timers or callbacks after cleanup

### F. No provider-specific error handling tests
- No test verifies behavior when Anthropic returns a 529 (overloaded) error
- No test verifies behavior when OpenAI returns rate-limit headers
- No test verifies Google provider's handling of safety filter blocks

### G. No tests for the web game client
- `src/web/public/game/` JavaScript files have no tests at all
- `dashboard.html` has no behavioral tests
- Frontend game fixtures (`fixtures.js`) are tested structurally but not functionally
